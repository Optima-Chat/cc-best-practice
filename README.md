# AI Native 开发最佳实践

> 徐昊 | Optima AI 创始人
> 2026-03-06 神州数码技术分享

Optima AI 做跨境电商 AI 平台，核心团队 3 个人，6 个月写了 120 万行代码，30 个仓库，8000 多次 commit。全程用 Claude Code 开发。

这个代码量和系统复杂度，按传统模式至少需要 20-30 人的团队。我们 3 个人做到了，靠的不是加班，而是从根本上改变了开发方式。

今天分享的不是理论，是我们踩了 6 个月坑之后总结出来的实战经验。分两部分：个人怎么用好 Claude Code，以及团队怎么围绕它协作。

一个核心观点先说在前面：**AI Native 开发最大的变化，是工程师的角色从"写代码的人"变成了"做决策的人"。** 学习、设计方案、审方案、理解原理、搭建工具链——这些才是人类该做的事。写代码交给 AI。

# 第一部分：个人开发

## 1. [先学习、再做方案、再写代码](topics/01-learn-plan-code.md)

不要直接让 Claude Code 写代码。正确的流程是：**学习（1-2h）→ 技术方案 → 审方案 → 写代码**。先花时间从 Claude Code 那里学习领域知识，再让它写技术方案，你审完确认没问题后再动手。技术方案做透了，写代码很快——5000 行不过 30 分钟。

## 2. [选对工程基础设施](topics/02-engineering-infrastructure.md)

三个关键选择：**全部用 TypeScript**（Claude Code 最擅长的语言，静态类型检查能排除大量错误）、**尽量用 Monorepo**（共享 Context，开发效率最高）、**写好 CLAUDE.md**（项目知识库，决定了 Claude Code 对你项目的理解深度）。

## 3. [让 Claude Code 大量写自动化测试](topics/03-automated-testing.md)

Claude Code 写的代码你不确定对不对，但测试可以告诉你。让它写测试的成本极低——它很擅长——但带来的质量保障价值极高。我们的 [commerce-backend](https://github.com/Optima-Chat/commerce-backend) 有 1555 个测试用例，100% 通过率。**测试 + 静态类型检查**，是确保代码质量的两道防线。

## 4. [多开是效率的来源](topics/04-parallel-development.md)

单开 1 天最多大约 5,000 行代码，多开可以到 30,000 行。我一般 4 开，多的时候 8 开——一个仓库一个 Claude Code 实例，同时推进多个仓库的任务。前提是技术方案要做得足够细，Claude Code 才能独立执行。**多开本质上是对前置工作质量的兑现。**

## 5. [要充分理解代码原理](topics/05-understand-principles.md)

不能因为是 Claude Code 写的代码就不去了解。当代码量上来以后，解决复杂 bug 往往需要人类提供方向性的建议。你不需要知道每一行代码怎么写，但你必须知道系统是怎么运转的。

# 第二部分：团队协作

## 6. [CLI First：给 AI 和人类设计同一套工具](topics/06-cli-first.md)

所有系统能力都优先封装成 CLI 命令——CLI 是 AI 最自然的操作接口。我们把电商系统的所有操作封装成了 [commerce-cli](https://github.com/Optima-Chat/commerce-cli)（19 个模块、155+ 命令），AI 和人类用同一套命令。这也是架构演进的核心教训：从 MCP 协议迁移到 Agent + CLI，新增能力只需写一个命令。

## 7. [做团队自己的 Dev Skills](topics/07-dev-skills.md)

把系统的代码、日志、数据库、环境变量的访问方式都做到 Dev Skills 里。效果是很多功能开发、问题修复，Claude Code 可以全程全自动完成，不需要人为费劲提供信息。

## 8. [充分利用 GitHub Issues 和 PR](topics/08-github-issues-pr.md)

团队成员之间交流，就是互相发 Claude Code 写的 Issue。特别是跨仓库传递信息时，让 A repo 的 Claude Code 直接给 B repo 写 Issue——信息完整详实，比人工描述更准确。PR 也一样，充分记录变更信息和设计意图。

## 9. [让 Claude Code 做 Code Review](topics/09-code-review.md)

Claude Code 先审一遍——测试覆盖、逻辑漏洞、安全问题，它都能发现。人再审架构合理性和业务逻辑。两层审下来，质量比过去高很多，成本反而低了。

## 10. [充分利用 Claude Code 的文档能力](topics/10-documentation.md)

Claude Code 写文档详尽、准确、格式规范。对内写技术方案、架构文档，对外写对接文档、API 说明。过去团队不愿意写文档是因为成本太高，现在成本降到几乎为零，文档应该成为开发流程的标配。

## 11. [用 Claude Code 做 BI 分析](topics/11-bi-analysis.md)

传统 BI 告诉你"DAU 下降了 23%"，Claude Code 能告诉你**为什么**——从用户对话记录等非结构化数据中提炼洞察。定量 + 定性，才是完整的用户理解。**相当于每周做一次全面的用户调研，成本几乎为零。**

# 合作探讨

## 这对神州数码意味着什么

### 不限于电商——任何软件开发都适用

我今天举的例子都是电商系统，但 11 条实践本身跟业务领域无关。"先学习再做方案""让 AI 写测试""充分利用 Issues 和 PR"——这些是工程方法论，不是电商方法论。

对神州数码最直接相关的三个场景：

1. **内部平台和工具开发。** 内部 IT 系统、运营平台、管理工具。这类项目最适合试水——风险小、边界清晰、效果容易衡量。
2. **客户数字化转型项目交付。** AI Native 开发可以显著提高交付速度和质量，直接转化为项目利润率和客户满意度。
3. **遗留系统的维护和现代化。** Claude Code 很擅长理解老代码。让它先写测试覆盖老系统，再逐步重构，是风险最低的现代化路径。

### Java/Python 团队一样能用

11 条实践中，**8 条完全跟编程语言无关**：先学习再做方案、多开、理解原理、Issues/PR、Code Review、文档、Dev Skills、BI 分析——这些今天就能用。

剩下 3 条：

- **TypeScript 的静态类型检查 → Java 天然具备**，这个优势 Java 团队已经有了；Python 团队加 Type Hints + mypy 也能获得
- **Monorepo → Java 用 Maven 多模块或 Gradle 多项目**，同一个思路
- **CLAUDE.md → 任何语言、任何项目都可以写**，这是投入最小、见效最快的一步

我们选 TypeScript 是因为全栈统一语言的考虑，不是因为其他语言用 Claude Code 效果不行。

### 不同级别的工程师怎么切入

**资深工程师（架构师/Tech Lead）：** 生产力提升最大。他们已经具备决策能力，加上 AI 多开，一个人的产出相当于一个小团队。

**中级工程师：** 负责模块级别的开发。做方案、审方案的能力在实践中快速成长。AI 帮他们跳过大量重复编码，更快触达架构层面的思考。

**初级工程师：** 成长路径变了——从"理解系统、学做方案"开始，用 AI 作为导师和搭档。重点不是写代码的熟练度，而是系统思维和决策能力。成长速度反而更快。

**试点团队的建议画像：1 个资深 + 2-3 个中级 + 1-2 个初级，共 5-6 人。**

## 成本对比

### AI 工具成本

| 项目 | 月成本（人民币） |
|------|:---:|
| Claude Code 订阅（每人） | ~1,500 |
| 5 人试点团队 AI 工具总成本 | ~7,500 |

### 对比传统模式

| | AI Native 团队（5 人） | 传统团队（等效产出） |
|------|:---:|:---:|
| 人数 | 5 | 15-20 |
| 人力成本（月） | 25 万 | 75-100 万 |
| AI 工具成本（月） | 0.75 万 | 0 |
| **月总成本** | **~26 万** | **~75-100 万** |

> 人力成本按全成本（薪资+社保+办公+设备）每人约 5 万/月估算。产出倍率按 3-4 倍估算（保守值，我们实际达到了 7-10 倍）。

**AI 工具成本占人力成本不到 3%。** 即使生产力只提升 2 倍，ROI 也是显著正的。

真正的投入不是钱，是团队学习和适应新工作方式的时间——大约 2-4 周的磨合期。

## 合作提案

### 第一步：试点（3 个月）

**神州数码：**
- 选一个 5-6 人的团队（1 资深 + 2-3 中级 + 1-2 初级）
- 选一个新项目或独立模块（避免涉及敏感数据的核心系统）
- 给团队配好 Claude Code 订阅

**Optima AI：**
- 提供 AI Native 开发培训（2 天集中 + 每周 1 次远程辅导）
- 帮团队建立 CLAUDE.md、Dev Skills、CI 测试流程
- 协助做第一个项目的技术方案，示范"先学习再做方案"的完整流程

**3 个月后看数据：**
- 代码产出量 vs 同规模传统团队
- Bug 率和测试覆盖率
- 交付速度
- 团队成员的主观体验

### 第二步：推广（基于试点结果）

试点效果好的话：

- 扩展到 3-5 个团队
- 形成神州数码内部的 AI Native 开发规范
- 探索将 AI Native 开发能力作为数字化转型服务的一部分，向客户输出

### 合作模式

建议从咨询+培训开始，轻量级切入。具体合作形式和商务条款可以进一步讨论。

关键是先跑起来，用数据说话。3 个月试点的成本极低——几万块的 AI 工具费用加上 Optima 的辅导时间。但它能帮双方验证这个方向的可行性，也为后续更大规模的合作提供决策依据。
