# 贡献指南

感谢你考虑为 Claude Code 最佳实践项目做出贡献！

## 如何贡献

### 报告问题

如果你发现文档中的错误或有改进建议：

1. 前往 [Issues](../../issues)
2. 搜索是否已有相关问题
3. 如果没有，创建新 Issue
4. 清晰描述问题或建议

### 提交最佳实践

我们欢迎你分享使用 Claude Code 的经验！

#### 内容要求

- **真实性**: 基于实际使用经验
- **实用性**: 对其他用户有帮助
- **清晰性**: 易于理解和实践
- **完整性**: 包含足够的上下文和示例

#### 提交流程

1. **Fork 本仓库**

2. **创建分支**
   ```bash
   git checkout -b feature/my-best-practice
   ```

3. **添加内容**

   根据内容类型选择合适的目录：

   - 基础使用 → `docs/basics/`
   - 进阶技巧 → `docs/advanced/`
   - 实战案例 → `docs/examples/`
   - 最佳实践 → `docs/best-practices/`

4. **遵循文档规范**

   - 使用中文
   - Markdown 格式
   - 清晰的标题结构
   - 代码示例要有注释
   - 使用实际案例

5. **更新目录**

   在相应的 `README.md` 中添加指向新文档的链接

6. **提交更改**
   ```bash
   git add .
   git commit -m "添加: XXX 最佳实践"
   ```

7. **推送分支**
   ```bash
   git push origin feature/my-best-practice
   ```

8. **创建 Pull Request**

   - 清晰描述你的贡献
   - 说明为什么这个实践有价值
   - 如果有截图或示例，一并提供

## 文档编写规范

### 文件命名

- 使用小写字母
- 单词之间用短横线连接
- 使用 `.md` 扩展名

示例：
- ✅ `slash-commands.md`
- ✅ `project-structure.md`
- ❌ `SlashCommands.md`
- ❌ `project_structure.md`

### 文档结构

```markdown
# 标题

> 简短的描述（可选）

## 概述

简要说明这个主题...

## 详细内容

### 子主题 1

内容...

### 子主题 2

内容...

## 示例

实际的使用示例...

## 最佳实践

- 要点 1
- 要点 2

## 注意事项

需要注意的地方...

## 下一步

- 相关文档链接
```

### 代码示例

使用代码块并指定语言：

````markdown
```typescript
// TypeScript 示例
function example() {
  console.log('Hello');
}
```

```bash
# Shell 命令示例
npm install
```
````

### 链接规范

- **内部链接**: 使用相对路径
  ```markdown
  [进阶技巧](../advanced/README.md)
  ```

- **外部链接**: 使用完整 URL
  ```markdown
  [Claude Code 官方文档](https://docs.claude.com/claude-code)
  ```

### 强调和提示

使用 emoji 和标记增强可读性：

```markdown
✅ 推荐做法
❌ 不推荐做法
⚠️ 注意事项
💡 提示
📝 示例
🚀 快速开始
```

## 内容建议

### 适合的内容

- ✅ 实用的技巧和窍门
- ✅ 真实的使用案例
- ✅ 常见问题的解决方案
- ✅ 效率提升的方法
- ✅ 项目配置示例

### 不适合的内容

- ❌ 纯理论没有实践价值
- ❌ 过时的信息
- ❌ 重复已有内容
- ❌ 特定公司/项目的机密信息
- ❌ 推广性质的内容

## 审查标准

PR 会根据以下标准审查：

### 内容质量
- [ ] 准确性：信息正确无误
- [ ] 实用性：对用户有实际帮助
- [ ] 完整性：包含必要的上下文和示例
- [ ] 原创性：不重复现有内容

### 格式规范
- [ ] 遵循文档结构
- [ ] Markdown 格式正确
- [ ] 代码示例完整可运行
- [ ] 链接有效

### 语言表达
- [ ] 使用中文
- [ ] 表达清晰易懂
- [ ] 没有语法错误
- [ ] 术语使用准确

## 行为准则

### 尊重他人

- 友善、包容的交流
- 尊重不同的观点和经验
- 建设性的反馈

### 质量优先

- 确保内容准确
- 认真对待每个贡献
- 持续改进文档

### 开放协作

- 欢迎新手贡献
- 积极帮助他人
- 分享经验和知识

## 获取帮助

如果你在贡献过程中遇到问题：

1. 查看现有的 [Issues](../../issues)
2. 阅读其他文档作为参考
3. 创建新 Issue 提问
4. 在 [Discussions](../../discussions) 中讨论
